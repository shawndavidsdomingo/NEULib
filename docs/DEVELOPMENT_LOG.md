# NEU Library Portal: Comprehensive Development Log

This document tracks the evolution of the NEU Library Portal from its inception to the current institutional-grade deployment.

---

## 📅 Day 1: Project Inception & Core Terminal
**Goal**: Establish a functional visitor management system for the NEU Library.

### ✅ Updates
- **Initial Setup**: Configured Next.js 15 with Tailwind CSS and ShadCN UI.
- **Kiosk Implementation**: Created `TerminalView.tsx` supporting RFID simulation and Google Login (@neu.edu.ph).
- **Basic Firestore Schema**: Defined `students`, `admins`, and `library_logs`.
- **Identity Logic**: Implemented "Smart Traffic" detection to handle check-ins and check-outs via a single input field.

## 🛠 Bug Reports & Debugging History

### 🐞 Bug: Checkout Cut-off (Midnight Logic)
- **Status**: Fixed.
- **Issue**: Students were able to check out of sessions from previous days (e.g., checking out of a Mar 9 session on Mar 11), leading to incorrect duration logs.
- **Debugging**: Updated `TerminalView.tsx` to enforce a "Smart Cut-off". A student can now only "Check Out" if their latest log started **today**. Stale logs are ignored and automatically treated as "No Tap" sessions in all reports.

### 🐞 Bug: Missing `useMemo` Import
- **Status**: Fixed.
- **Issue**: `ReportModule.tsx` crashed due to a missing import of the `useMemo` hook from React.
- **Debugging**: Added `import { useMemo } from 'react'` to the top of the file.

### 🐞 Bug: Unrecognized Tag `<graduationcap>`
- **Status**: Fixed.
- **Issue**: React threw a console error because the `GraduationCap` component was invoked as a lowercase HTML tag.
- **Debugging**: Corrected the casing to `<GraduationCap />` in `SuperAdminDashboard.tsx` and `StaffDashboard.tsx`.

### 🐞 Bug: JSX Parsing Error (Student Dashboard)
- **Status**: Fixed.
- **Issue**: A missing closing parenthesis in the conditional rendering of the Recharts block caused a build failure.
- **Debugging**: Audited the JSX tree in `StudentDashboard.tsx` and restored structural integrity.

### 🐞 Bug: Background Visibility
- **Status**: Fixed.
- **Issue**: The `neulibrary.jpg` background was being covered by solid white containers or was too dark.
- **Debugging**: Updated `globals.css` to use a light `rgba(255, 255, 255, 0.85)` overlay and ensured the `institutional-bg` div was placed correctly at the root of `layout.tsx`.

---

## 🧠 Smart Logic: "No Tap" (Missed Checkout) Detection
Implemented an automated system to handle stale sessions.
- **Logic**: If a session has a `checkInTimestamp` from a previous day but no `checkOutTimestamp`, it is dynamically identified as **"No Tap"**.
- **Execution**: The system "marks" these sessions automatically at 12:00 AM of the following day, displaying a red status badge in all reports and logs. Students tapping at the kiosk on a subsequent day will start a new session instead of closing the stale one.

---

## 🔒 Security Protocol Update
- **Manual Termination Restricted**: Removed the ability for Admins to "Time Out" students manually. This ensures that attendance logs are strictly generated by physical student interactions at the Kiosk, maintaining data integrity for institutional reporting.
